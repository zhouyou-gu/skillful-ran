# Promotion Rubric

- `keep`: compact, sanitized, reusable, and already shaped like a script, reference, or tiny example
- `rewrite`: useful material that still needs normalization, renaming, trimming, or relocation into the right skill folder
- `drop`: raw logs, captures, binaries, secrets, host transcripts, or clutter with no clear future value

Hard gates before tracked promotion:

- clean git worktree
- valid namespace and category
- required tracked files generated
- eval pack present
- compactness preserved
